Firebase 설정

아래 Firebase 설정값은 index.html 코드 상단의 FIREBASE_CONFIG에 직접 넣어 GitHub에 올리는 방식입니다. 화면에서 임시 입력해 테스트할 수도 있지만, 근로자 QR 접속까지 되려면 코드에 반영되어 있어야 합니다.


    

  


  
초보 설치가이드
1) Firebase 프로젝트 만들기
Firebase 콘솔에서 프로젝트를 만들고 웹앱을 추가합니다. 표시되는 firebaseConfig 값을 복사합니다.
2) Authentication 익명 로그인 켜기
Authentication → Sign-in method → Anonymous를 사용 설정합니다. 근로자는 로그인 화면을 보지 않습니다.
3) Firestore Database 만들기
Firestore Database를 만들고 컬렉션 이름은 자동으로 workerOpinions를 사용합니다.
4) 사진 저장 방식
이 버전은 Firebase Storage를 사용하지 않습니다. 사진은 브라우저에서 자동 압축 후 Firestore 문서 안에 저장됩니다. 따라서 Storage 설정 없이도 사진 첨부를 테스트할 수 있습니다.
5) 보안 규칙
초기 테스트용 규칙은 아래처럼 사용합니다. 운영 전에는 관리자 로그인 방식으로 강화하는 것이 좋습니다.
Firestore rules:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /workerOpinions/{docId} {
      allow create, read, update: if request.auth != null;
      allow delete: if false;
    }
  }
}
6) QR 생성
GitHub Pages 주소에서 QR 안내문 → QR 생성 → A4 안내문 출력. QR 주소는 ?mode=form이 붙은 GitHub Pages 주소입니다.
