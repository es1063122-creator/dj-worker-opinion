동진 종사자 의견청취 QR 설치가이드

구글시트 · Apps Script · SHEET_ID · setupSheets · 웹앱 배포 URL 저장까지 초보자용 순서 안내

① 구글시트 만들기 ② Apps Script 만들기 관리화면으로 돌아가기

📖 초보자용 최초 설치가이드

아래 순서대로 하면 됩니다. 이 페이지는 현장 관리자나 본사 담당자가 처음 설치할 때 옆에 켜두고 따라 하는 용도입니다.

전체 흐름
구글시트는 데이터 저장소, Apps Script는 서버/접수폼, 이 웹페이지는 본사 관리자 화면입니다. 설치가 끝나면 근로자는 QR로 의견을 제출하고, 본사는 전현장 현황을 확인할 수 있습니다.

1구글시트 만들기

무엇을 하는 단계? 접수된 의견, 현장QR, 반기점검 자료가 저장될 엑셀 같은 파일을 만드는 단계입니다.

  1. 상단 최초 설정 탭에서 ① 구글시트 만들기를 누릅니다.
  2. 새 구글시트가 열리면 왼쪽 위 제목을 동진_종사자의견청취_관리대장으로 바꿉니다.
  3. 브라우저 주소창에서 아래 부분을 복사해야 합니다.
https://docs.google.com/spreadsheets/d/여기가_SHEET_ID/edit#gid=0

입력할 내용: 이 단계에서는 제목만 바꾸면 됩니다. 시트 안의 표 제목은 나중에 setupSheets()가 자동으로 만듭니다.

2Apps Script 만들기

무엇을 하는 단계? QR 제출화면과 본사 관리자 화면이 구글시트에 데이터를 저장/조회할 수 있게 하는 서버를 만드는 단계입니다.

  1. 상단 최초 설정 탭에서 ② Apps Script 만들기를 누릅니다.
  2. 새 Apps Script 화면이 열리면 프로젝트 이름을 동진 종사자의견청취 QR 서버로 바꿉니다.
  3. 기본으로 보이는 Code.gs 안의 내용을 전부 지웁니다.

입력할 내용: 프로젝트 이름만 바꿉니다. 코드는 다음 단계에서 붙여넣습니다.

3설치코드 복사/붙여넣기

무엇을 하는 단계? 미리 만들어둔 서버 코드를 Apps Script에 넣는 단계입니다.

  1. 상단 최초 설정 탭에서 ③ 설치코드 복사를 누릅니다.
  2. Apps Script 화면으로 돌아갑니다.
  3. 비어 있는 Code.gs에 붙여넣기 합니다. 단축키는 Ctrl + V입니다.
  4. 붙여넣은 뒤 Ctrl + S 또는 저장 아이콘을 누릅니다.
복사가 안 되면 새로 열린 설치코드 창에서 Ctrl+A → Ctrl+C 하거나, 압축파일 안의 apps_script.gs 파일을 직접 열어서 전체 복사하면 됩니다.

4SHEET_ID 입력

무엇을 하는 단계? Apps Script가 어느 구글시트에 저장할지 알려주는 단계입니다.

  1. Apps Script 코드 맨 위쪽에서 SHEET_ID 또는 CONFIG.SHEET_ID라고 적힌 부분을 찾습니다.
  2. 1단계에서 복사한 구글시트 ID를 따옴표 안에 붙여넣습니다.
const CONFIG = { SHEET_ID: "구글시트_ID를_여기에_붙여넣기", ADMIN_PIN: "5183" };

입력 예시:

SHEET_ID: "1AbCDeFgHIjkLmNoPqRstUvWxYz1234567890"

주의: 주소 전체를 넣으면 안 됩니다. 반드시 /d/와 /edit 사이의 긴 글자만 넣으세요.

5setupSheets() 실행

무엇을 하는 단계? 구글시트 안에 필요한 탭과 표 제목을 자동으로 만드는 단계입니다.

  1. Apps Script 상단의 함수 선택 드롭다운에서 setupSheets를 선택합니다.
  2. 실행 버튼을 누릅니다.
  3. 권한 승인 창이 나오면 본인 구글계정을 선택합니다.
  4. 경고 화면이 나오면 고급 → 프로젝트로 이동 → 허용을 누릅니다.
  5. 실행이 끝나면 구글시트에 의견접수 / 현장QR / 반기점검 탭이 생겼는지 확인합니다.

정상 결과: 구글시트 하단에 의견접수, 현장QR, 반기점검 시트가 생기면 성공입니다.

6웹앱 배포 URL 저장

무엇을 하는 단계? QR 제출화면과 본사 관리자 화면이 사용할 인터넷 주소를 만드는 단계입니다.

  1. Apps Script 오른쪽 위 배포를 누릅니다.
  2. 새 배포를 누릅니다.
  3. 유형 선택에서 웹 앱을 선택합니다.
  4. 설명은 동진 의견청취 QR v1로 입력합니다.
  5. 실행 사용자는 나로 선택합니다.
  6. 액세스 권한은 모든 사용자로 선택합니다.
  7. 배포를 누르고 나온 웹 앱 URL을 복사합니다.
  8. 이 관리화면 상단의 Google Apps Script 배포 URL에 붙여넣고 설정 저장을 누릅니다.
https://script.google.com/macros/s/배포ID/exec

관리자 PIN: 기본값은 코드에서 설정한 5183을 입력하면 됩니다. 나중에 바꾸려면 Apps Script 코드의 ADMIN_PIN 값을 수정하면 됩니다.

7설치 후 테스트

  1. 상단에 Apps Script 배포 URL과 관리자 PIN을 입력하고 설정 저장을 누릅니다.
  2. 현장 QR 관리 탭으로 이동합니다.
  3. 현장명에 테스트 현장, 관리자에 김대홍을 입력합니다.
  4. QR 생성 → 현장QR 등록/수정을 누릅니다.
  5. 제출화면 열기를 눌러 테스트 의견을 1건 제출합니다.
  6. 본사 관리자 탭에서 본사 현황 불러오기와 의견 목록 불러오기를 눌러 접수 여부를 확인합니다.

성공 기준: 본사 관리자 화면에 테스트 의견이 보이면 설치 완료입니다.

자주 실수하는 부분

  • SHEET_ID에 주소 전체를 넣는 경우 → /d/와 /edit 사이 글자만 넣어야 합니다.
  • setupSheets를 실행하지 않은 경우 → 시트 탭이 없어서 저장이 안 됩니다.
  • 웹앱 액세스 권한을 본인만으로 한 경우 → 근로자가 QR 화면을 열 수 없습니다.
  • Apps Script 수정 후 재배포하지 않은 경우 → 예전 코드가 계속 실행될 수 있습니다. 수정 후에는 배포 관리를 확인하세요.
  • 관리자 PIN 불일치 → Apps Script 코드의 ADMIN_PIN과 화면에 입력한 PIN이 같아야 합니다.
